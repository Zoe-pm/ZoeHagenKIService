Testplan (manuell, vor Go-Live)
	1.	Cold Load (Desktop/Mobil): Seite laden → kein Replit-Default sichtbar. UI zeigt unseren Chat.
	2.	Happy Path: Frage senden → Antwort von n8n ≤ 3s (oder wie vereinbart).
	3.	Webhook down (simuliert): n8n stoppen oder URL temporär falsch → Banner erscheint, kein Replit-Default.
	4.	CORS Test: Zugriff von Staging-Domain → OK; von fremder Domain → geblockt.
	5.	Republish: „Republish“ klicken → Deploy bleibt Static. (Screenshot + Deploy-Log sichern.)
	6.	Refresh/Retry: Nach Fehler „Erneut versuchen“ → wenn n8n wieder up, klappt’s.

Akzeptanzkriterien (DoD)
	•	Nie erscheint der Replit-Default-Bot – weder bei Erfolg noch bei Fehlern.
	•	Fehlerfälle zeigen unser Banner + kein ungewollter Fallback.
	•	Zwei aufeinanderfolgende Publishes bleiben Static (Nachweis: Screenshots/Logs).
	•	Health-Check dokumentiert, Alert getestet (Test-Alert vor Abschluss).
	•	Testplan Punkte 1–6 bestehen.

⸻

Rollback-Plan
	•	Falls nach Publish Fehlerquote > 10% in 30 Min:
	•	Sofort Rollback auf letzte stabile Static-Version (Link/ID anlegen).
	•	Fehlerursache isolieren (Webhook, CORS, Timeout, Frontend).
	•	Erst nach erfolgreichem Test erneut publishen.
